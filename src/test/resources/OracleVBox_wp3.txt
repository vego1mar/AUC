          <div id="wikipage"><h1 id="DownloadVirtualBoxforLinuxHosts">Download VirtualBox for Linux Hosts</h1>
<p>
<strong>Note:</strong> The package architecture has to match the Linux kernel architecture, that is, if you are running a 64-bit kernel, install the appropriate AMD64 package (it does not matter if you have an Intel or an AMD CPU). Mixed installations (e.g. Debian/Lenny ships an AMD64 kernel with 32-bit packages) are not supported. To install VirtualBox anyway you need to setup a 64-bit chroot environment.
</p>
<p>
The VirtualBox base package binaries are released under the terms of the <a class="wiki" href="/wiki/GPL">GPL version 2</a>.
</p>
<p>
Please choose the appropriate package for your Linux distribution.

</li><li><a class="ext-link" href="https://download.virtualbox.org/virtualbox/6.0.4/virtualbox-6.0_6.0.4-128413~Ubuntu~xenial_amd64.deb"><span class="icon"> </span>Ubuntu 16.04</a>

</li><li><a class="ext-link" href="https://download.virtualbox.org/virtualbox/6.0.4/virtualbox-6.0_6.0.4-128413~Debian~stretch_amd64.deb"><span class="icon"> </span>Debian 9</a>

</p>
<p>
You might want to compare the checksums to verify the integrity of downloaded packages.
<em>The SHA256 checksums should be favored as the MD5 algorithm must be treated as insecure! </em>
<ul><li><a href="/download/hashes/6.0.4/SHA256SUMS">SHA256 checksums</a>, <a href="/download/hashes/6.0.4/MD5SUMS">MD5 checksums</a>
</li></ul>
</p>
<h3 id="OracleLinux">Oracle Linux</h3>
<p>
Users of Oracle Linux 6 and 7 can use the Oracle Linux yum <a class="ext-link" href="http://yum.oracle.com/"><span class="icon"> </span>repository</a> and enable the <a class="ext-link" href="http://yum.oracle.com/repo/OracleLinux/OL6/developer/x86_64/index.html"><span class="icon"> </span>ol6_developer</a> channel for Oracle Linux 6 or the <a class="ext-link" href="http://yum.oracle.com/repo/OracleLinux/OL7/developer/x86_64/index.html"><span class="icon"> </span>ol7_developer</a> channel for Oracle Linux 7. After that, do
</p>
<pre class="wiki">yum install VirtualBox-6.0
</pre><p>
to get the latest maintenance release of VirtualBox 6.0.x installed.
</p>
<h3 id="Debian-basedLinuxdistributions">Debian-based Linux distributions</h3>
<p>
Add the following line to your <tt>/etc/apt/sources.list</tt>. According to your distribution, replace '<tt>&lt;mydist&gt;</tt>' with '<tt>artful</tt>', '<tt>zesty</tt>', '<tt>yakkety</tt>', '<tt>xenial</tt>', '<tt>trusty</tt>', '<tt>stretch</tt>', '<tt>jessie</tt>', or '<tt>wheezy</tt>' (older versions of VirtualBox supported different distributions):
</p>
<pre class="wiki">deb https://download.virtualbox.org/virtualbox/debian &lt;mydist&gt; contrib
</pre><p>
The Oracle public key for apt-secure can be downloaded
</p>
<ul><li><a href="https://www.virtualbox.org/download/oracle_vbox_2016.asc">here</a> for <strong>Debian 8</strong> ("Jessie") / <strong>Ubuntu 16.04</strong> ("Xenial") and later
</li><li><a href="https://www.virtualbox.org/download/oracle_vbox.asc">here</a> for older distributions.
</li></ul><p>
You can add these keys with
</p>
<pre class="wiki">sudo apt-key add oracle_vbox_2016.asc
sudo apt-key add oracle_vbox.asc
</pre><p>
or combine downloading and registering:
</p>
<pre class="wiki">wget -q https://www.virtualbox.org/download/oracle_vbox_2016.asc -O- | sudo apt-key add -
wget -q https://www.virtualbox.org/download/oracle_vbox.asc -O- | sudo apt-key add -
</pre><p>
The key fingerprint for oracle_vbox_2016.asc is
</p>
<pre class="wiki">B9F8 D658 297A F3EF C18D  5CDF A2F6 83C5 2980 AECF
Oracle Corporation (VirtualBox archive signing key) &lt;info@virtualbox.org&gt;
</pre><p>
The key fingerprint for oracle_vbox.asc is
</p>
<pre class="wiki">7B0F AB3A 13B9 0743 5925  D9C9 5442 2A4B 98AB 5139
Oracle Corporation (VirtualBox archive signing key) &lt;info@virtualbox.org&gt;
</pre><p>
<em>(As of VirtualBox 3.2, the signing key was changed. The old Sun public key for apt-secure can be downloaded <a class="ext-link" href="https://download.virtualbox.org/virtualbox/debian/sun_vbox.asc"><span class="icon"> </span>here</a>.)</em>
</p>
<p>
To install VirtualBox, do
</p>
<pre class="wiki">sudo apt-get update
sudo apt-get install virtualbox-6.0
</pre><p>
Replace <tt>virtualbox-6.0</tt> by <tt>virtualbox-5.2</tt> to install the latest VirtualBox 5.2 build.
</p>
<p>
What to do when experiencing <strong>The following signatures were invalid: BADSIG ...</strong> when refreshing the packages from the repository?
</p>
<pre class="wiki"># sudo -s -H
# apt-get clean
# rm /var/lib/apt/lists/*
# rm /var/lib/apt/lists/partial/*
# apt-get clean
# apt-get update
</pre><h3 id="RPM-basedLinuxdistributions">RPM-based Linux distributions</h3>
<p>
We provide a yum/dnf-style repository for Oracle Linux/Fedora/RHEL/openSUSE. All .rpm packages are signed. The Oracle public key for rpm can be downloaded <a href="https://www.virtualbox.org/download/oracle_vbox.asc">here</a>. You can add this key <em>(not normally necessary, see below!)</em> with
</p>
<pre class="wiki">sudo rpm --import oracle_vbox.asc
</pre><p>
or combine downloading and registering:
</p>
<pre class="wiki">wget -q https://www.virtualbox.org/download/oracle_vbox.asc -O- | rpm --import -
</pre><p>
The key fingerprint is
</p>
<pre class="wiki">7B0F AB3A 13B9 0743 5925  D9C9 5442 2A4B 98AB 5139
Oracle Corporation (VirtualBox archive signing key) &lt;info@virtualbox.org&gt;
</pre><p>
After importing the public key, the package signature can be checked with
</p>
<pre class="wiki">rpm --checksig PACKAGE_NAME
</pre><p>
<em>Note that importing the key is not necessary for yum users (Oracle Linux/Fedora/RHEL/CentOS) when using one of the virtualbox.repo files from below as yum downloads and imports the public key automatically! Zypper users should run
</em></p>
<pre class="wiki">zypper refresh
</pre><p>
<em>
</em></p>
<p>
The package signature is checked by yum/dnf/zypper as well:
</p>
<ul><li>Users of <strong>Oracle Linux</strong> / <strong>RHEL</strong> can add <a class="ext-link" href="https://download.virtualbox.org/virtualbox/rpm/el/virtualbox.repo"><span class="icon"> </span>the Oracle Linux repo file</a> to <em>/etc/yum.repos.d/</em>.
</li><li>Users of <strong>Fedora</strong> can add <a class="ext-link" href="https://download.virtualbox.org/virtualbox/rpm/fedora/virtualbox.repo"><span class="icon"> </span>the Fedora repo file</a> to <em>/etc/yum.repos.d/</em>.
</li><li>Users of <strong>openSUSE</strong> can add <a class="ext-link" href="https://download.virtualbox.org/virtualbox/rpm/opensuse/virtualbox.repo"><span class="icon"> </span>the openSUSE repo file</a> to <em>/etc/zypp/repos.d/</em>.
</li></ul></div>
